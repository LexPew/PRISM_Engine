🧩 Engine Utility Class Structure — Meyers Singleton Pattern

In this engine, each high-level utility class (e.g. Renderer, Window, SceneManager, AudioManager) should follow the Meyers Singleton pattern.

This approach ensures:

Only one global instance exists.

Thread-safe lazy initialization.

No manual memory management.

Cleaner and more modern C++ design.

✅ General Structure
#pragma once
#include <stdexcept>

class NAME
{
private:
    // Private constructor prevents direct instantiation
    NAME();


public:
    // Meyers Singleton access function
    static NAME& Get()
    {
        static NAME instance; // Created once, thread-safe in C++11+
        return instance;
    }

    // Example methods
    void Initialize();
    void Update();
    void Shutdown();
};
